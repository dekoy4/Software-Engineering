# SRS: Калькулятор финансового планирования
## Введение

### Область действия (Scope)

Система представляет собой консольное Python-приложение для ввода доходов и расходов, расчета баланса и прогнозирования бюджета на месяц. Не включает GUI, интеграцию с банками или мультипользовательский режим. Выгоды: контроль личных финансов, предупреждение о превышении лимитов, простота использования без установки.

### Определения, акронимы и сокращения

- **SRS**: Software Requirements Specification (Спецификация требований ПО).
- **CSV**: Comma-Separated Values (формат табличных данных).
- **UI**: User Interface (пользовательский интерфейс).

## Общее описание

### Взаимодействие с продуктом (Product Perspective)

Независимое консольное приложение. Взаимодействует только с локальными CSV-файлами для хранения данных. Не требует внешних сервисов или баз данных.

### Функции продукта (Product Functions)

- Ввод доходов и расходов по категориям.
- Расчет текущего баланса и прогноза.
- Установка и контроль лимитов.
- История операций в табличном виде.


### Пользователи и характеристики (User Classes and Characteristics)

- **Пользователь**: Взрослый человек с базовыми навыками работы в терминале. Знает суммы доходов/расходов. Права: полный доступ к данным.


### Окружение и ограничения (Operating Environment)

- ОС: Windows/Linux/macOS.
- Python 3.10+, стандартная библиотека + pandas.
- Терминал: PowerShell/cmd/bash.


### Предположения и зависимости

- Стабильное питание ПК.
- Зависимость: pandas для табличных операций.


## Детальные требования

### Функциональные требования

- **FR-001**: Ввод дохода (сумма, дата, источник: зарплата/фриланс).
- **FR-002**: Ввод расхода (сумма, дата, категория: еда/жилье/транспорт).
- **FR-003**: Расчет баланса = доходы - расходы.
- **FR-004**: Прогноз остатка на конец месяца (линейная экстраполяция).
- **FR-005**: Установка лимитов по категориям с предупреждением.
- **FR-006**: Просмотр истории (неделя/месяц) в таблице.
- **FR-007**: Сохранение/загрузка данных из `budget.csv`.


### Нефункциональные требования

#### Требования к производительности

Время расчета < 0.5 сек для 1000 записей.

#### Требования к надежности

Автосохранение каждые 5 операций. Восстановление при сбое.

#### Требования к безопасности

Локальное хранение, без сетевых протоколов. Защита от некорректного ввода.

#### Требования к удобству использования

Меню с цифрами 1-7. Помощь по командам. Обучение < 2 мин.

#### Требования к сопровождаемости

Модульный код, docstrings, 80% покрытие тестами (unittest).

#### Требования к портируемости

Работает на Python 3.10-3.12 без изменений.

## Требования к интерфейсам

### Пользовательские интерфейсы (UI)

Консольное меню:

```
1. Добавить доход
2. Добавить расход  
3. Показать баланс
4. Установить лимит
5. История
6. Сохранить и выйти
```

## Требования к данным

### Логическая модель данных

```
Transactions:
- id: int
- type: 'income'|'expense'
- amount: float
- date: datetime
- category: str (еда/жилье/...)
- description: str
```


### Требования к хранению

CSV-файл `budget.csv`, до 10 000 записей (5 МБ).

### Требования к целостности

- Сумма > 0.
- Дата в формате ДД.ММ.ГГГГ.
- Категория из предустановленного списка.
